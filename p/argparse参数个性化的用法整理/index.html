使用前要实例化一个ArgumentParser对象

parser = argparse.ArgumentParser(description="这是ArugementParser中的description")

parser.parse_args()

这上面当你运行

python test.py -h

注意要加上-h参数，就是出现主要的decsription中的str

parser = argparse.ArgumentParser()
parser.add_argument("test")
args = parser.parse_args()
print("输出的内容是%s" % args.test)

这个其实就是添加了cmd中一个位置，将获取到的值存入到test这个变量中 要取变量的值先通过parse_args()获取到args对象

比如我运行

python test.py hello

则会返回 输出的内容是hello

但是运行 python test.py 则会提示缺乏argument参数，而且会提示那个参数命为test，即代码中写的， 那么用这个来可以提示用户缺乏了啥参数，即把test换成描述性语言

运行 python test.py hello world 则会返回 多了个参数，因为代码中只定义了一个占位

这里怎么为这个参数添加描述呢，就是说怎么让别人运行-h时能知道这个参数代表啥呢？

add_argument()中的help参数可以帮忙

parser = argparse.ArgumentParser()
parser.add_argument("test",help="这是test的代表意义")
args = parser.parse_args()
print("输出的内容是%s" % args.test)

运行python test.py -h 时就可以展示这个信息了

add_argument()中的参数type可以指定参数类型,没有指定则会默认为str

传入不符合的则会报错

parser = argparse.ArgumentParser()
parser.add_argument("test",typ e=int)
args = parser.parse_args()
print(args)

可选参数，这里觉得可以起到一个定位参数的作用

add_argument参数"-" / “–”；在参数前加上前缀–，即意味着这个参数是可选参数,和之前的”里面不加-是不一样的含义

– / - 表示参数可选，用户给不给值无所谓，而不加- / – 则表示这个参数必选要用

parser = argparse.ArgumentParser()
#“”是必须的
parser.add_argument("-t")
args = parser.parse_args()
print("%s" % args)

运行python test.py不会报错，只是print那里会用None替代

运行python test.py -t test 完美！！

add_argument参数action=“store_true”

parser = argparse.ArgumentParser()
parser.add_argument("-t",action="store_true")
args = parser.parse_args()
print("%s" % args)

运行python test.py -t 只要使用了-t 的值则会True,但是运行python test.py -t test 则会报错，也就是说定义了action=“store_true”，就不能从命令行传值,实现这一目的的方法是将action参数的值指定为 “store_true”

通过这个特点，就可以实现使用某一个特定功能这个目的

add_argument中的“-”和“–”指定参数是一样的，但是这里有个比较巧妙的地方是：

parser = argparse.ArgumentParser(description="这是主要描述")
parser.add_argument("-t","--test",help="这是代码中test（-t）的意义")
parser.add_argument("-d","--dest",help="这是代码中dest（-d）的意义")
args = parser.parse_args()

上面add_argument “-” ”–“ 这样写的好处在于–后可以直接指定意义，方便写代码，如下

print(args.test) #等同于args.t
print(args.dest)

add_argument中的参数choices可以为参数传入的值设定一个范围,用户传入的值不在范围内则报错,注意用户上传然后系统读取到的是str类型，所以如果choices是int类型的话，要指定type=int,否则会报错

parser = argparse.ArgumentParser()
parser.add_argument("-t","--test",type=int,choices=[1,2,3])
args = parser.parse_args()
print(args.test)

add_argument中的参数action=“count"可以计算出用户使用了这个参数的次数,但是指定了action=“count"的话用户就不能传值了，否则会报错

parser = argparse.ArgumentParser()
parser.add_argument("-t","--test",action="count",help="这是test的help")
arg = parser.parse_args()
print(arg.test)

add_argument()中的参数default可以指定参数的默认值，没设定默认值时默认值为None

设置冲突参数 就是说当一些参数只能选择一个使用时，可以用方法add_mutually_exclusive_group()

parser = argparse.ArgumentParser()
group = parser.add_mutually_exclusive_group()
group.add_argument("-t")
group.add_argument("-d")

args = parser.parse_args()

下面是对ArgumentParser方法中参数的说明：

prog中可以赋予运行程序出现的语句,默认为sys.argv[0]就是你脚本的名字,就是程序此时会变成你自定义的名字

parser = argparse.ArgumentParser()
parser = argparse.ArgumentParser(prog="这是测试")
parser.print_help()

usage可以修改cmd运行程序中第二个出现的字体, 如果要传递prog中的值，则需要写成 %(prog)s

parser = argparse.ArgumentParser(prog="这是测试",usage="%(prog)s [usage中的字]")
parser.print_help()

description则可以添加主要描述，会换行

parser = argparse.ArgumentParser(prog="这是测试",usage="%(prog)s [usage中的字]",description="这是主要描述")
parser.print_help()

epilog可以赋予结束语的描述

parser = argparse.ArgumentParser(
    prog="这是测试",usage="%(prog)s [usage中的字]",
    description="这是主要描述",
    epilog="这是结束语")
parser.print_help()

parents这个没看懂有什么用

ormatter_class 这个参数可以指定展示文字的格式

prefix_chars可以修改运行程序时候前面的-,改成自己需要的样式，如，想要运行python test.py +test 123

parser = argparse.ArgumentParser(prefix_chars="+")
parser.add_argument("+t")
parser.add_argument("+d")

args = parser.parse_args()
print(args.t,args.d)

fromfile_prefix_char=‘@’ 从文件中读取命令参数 有一个文件为args.txt 注意文件中的参数和值要换行 在parse_args(["@args.txt”]) 类似这样传入文件命 命令行参数从文件中读取

parser = argparse.ArgumentParser(fromfile_prefix_chars='@')
parser.add_argument('-f')
parser.parse_args(['-f', 'foo', '@args.txt'])

exit_on_error=False 可以捕获用户传入的值不符合要求,而不会被argparsr中断程序

错误类型为argparse.ArgumentError,只存在Python3.9.。。。

parser = argparse.ArgumentParser(exit_on_errors=False)
parser.add_argument("-t","--t",type=int)

try:
    parser.parse_args("--t a".split())
except argparse.ArgumentError:
    print("nonon!!")

add_argument方法中的参数说明

name or flags其实就是赋予参数名

parser = argparse.ArgumentParser()
parser.add_argument("-t","--t")#一般会赋予短名和长名

action则是用来对参数的值进行互动的

  • 默认为action = “store” 即存储值
  • action = “store_const” 这是存储const的值；用户可以传值但是没有用，改变不了这个const的值
parser.add_argument("--t",action="store_const",const=20)
print(parser.parse_args(['--t']))
  • action = “store_true” / action = “store_false”,可以理解为一种开关，只有用户使用了这个参数，才会返回Ture,否则会false
parser.add_argument("--t",action="store_true")
print(parser.parse_args().t)

result:

python test.py -t 是Ture
python test.py 是False
python test,py -t 123 报错，所以是不能传值的
  • action = “append”,这样写用户可以 python test.py -t 123 -t 123 传入多个t的值，且会被放到一个list中
parser.add_argument("-t",action="append")
print(parser.parse_args().t)

这样写的话，可以运行python test.py -t 123 -t 456 但是只用456值会显示,就是说它会用最新的值取覆盖旧的值

parser.add_argument("-t")
print(parser.parse_args().t)
  • action = “append_const”,这样会将const定义的的值传给定义的参数中，一般在多个参数需要在同一列表中存储常数时会有用
  • action = “count”,这个可以用来计算一个参数出现的次数
parser.add_argument("-t",action="count",default=0)
print(parser.parse_args().t

运行python test.py -t -t 和 python test,py -tt 是一样的效果,python test.py -t 123 会报错，就是说定义了action=“count"的话用户是不能传值的

  • action=“version”,help参数则是用来添加版信息,action定义了version，则后面要说明version的str
parser = argparse.ArgumentParser(prog="TEST")
parser.add_argument("-t",action="version",version=" %(prog)s 2.0")
args = parser.parse_args()
print("输出的内容是%s" % args.t)

nargs,如果等于一个数字起到一个指定参数用户可以传入多少个值，然后程序会将这些值存在一个list中, 如果等于”*” “+” 则会将用户所有参数传入的值放在同一个list中,如果等于"?" 则运行用户直接提供一个文件，注意程序中记得将type = argparse.FileType(“w”)

parser = argparse.ArgumentParser()
parser.add_argument("-t","--test",nargs=3)
print(parser.parse_args().test)

运行python test.py -t 123 456 789这样才可以 python test.py -t 123 456 和 python test.py -t 123 不行

parser = argparse.ArgumentParser()
parser.add_argument("-t","--test",nargs="+")
parser.add_argument("-dt","--dtest",nargs="+")
print(parser.parse_args())

python test.py -t 123 -dt 456 则会将 123 456 放在一个list中

type,可以定义用户传入数据的类型

choic,可以限制用户输入参数的值范围以及选项

requeired=True,可以将一个可选参数变为必选

help,则是添加帮助信息,具体表现为用户运行python test.py -h 时显示的参数信息

parser =argparse.ArgumentParser()
parser.add_argument("-t",help="123")
parser.add_argument("-d",help="456")
#要有下面这句话 python test.py -h 才有用
parser.parse_args()

parser = argparse.ArgumentParser(prog='frobble')
parser.add_argument('bar', nargs='?', type=int, default=42,
                    help='the bar to %(prog)s (default: %(default)s)')
parser.print_help()

由这个可以看出如果要在方法内调用内部参数的值，则要写成 %(参数名)s

metavar,在显示帮助信息时，会将参数名字自动变成大学的，如-t -T,使用metavar 可以自定义-T名字

arser = argparse.ArgumentParser()
parser.add_argument("-t","--test",metavar="666")
parser.parse_args()

usage: test.py [-h] [-t 666]

optional arguments:
  -h, --help          show this help message and exit
  -t 666, --test 666

dest,实则是修改属性名，系统会自动将–test 或者 -test 生成这个dest的值 ,可以修改

parser = argparse.ArgumentParser()
parser.add_argument("-t","--test",dest="dddtest")
parser.parse_args()

usage: test.py [-h] [-t DDDTEST]

optional arguments:
  -h, --help            show this help message and exit
  -t DDDTEST, --test DDDTEST

参数和参数值在程序中的赋予 一般写成这样 ["–test",“value”] 对于长参数(–开头)可以写成这样 ["–test=value"] 对于短参数（-开头）可以写成这样 ["-tvalue"] 另外对于短参数的话，可以使用["-xyzT"] 连接起来 x,y,z是参数，T是值，这里的结果是x=True,y=True,z=T

parser = argparse.ArgumentParser()
parser.add_argument("-t",action="store_true")
parser.add_argument("-d")
parser.parse_args(["-td1"])

参数缩写,在程序内部是可以不写全参数名字的

parser = argparse.ArgumentParser()
parser.add_argument("-test")
parser.add_argument("-dtest")
a = parser.parse_args("-tes 123".split())
parser.parse_args("-dtes 456".split())
print(a)

程序内部赋值和用户赋值是不同的,就是说用户运行 python test.py -t 123 这是用户赋予-t参数的值123,而在程序内部则是 这样赋值的 parse_arg(["-t123"]) 等写法
